# VSLLM 签到 + 今日任务 + 抽奖控制台

一个面向 Windows 的 VSLLM 日常自动化工具，支持每日签到、今日任务、翻牌抽奖、余额读取和冷却守护。

> 本项目仅用于个人学习和个人账号管理。使用前请确认你了解并遵守目标网站的服务条款、频率限制和账号规则。

## 功能

- Windows 图形控制台，常用操作都可通过按钮完成。
- 首次登录使用本机 Chrome 或 Microsoft Edge，不需要手动复制 Cookie。
- 登录态仅保存在本机 `.auth/vsllm-profile`。
- 自动检查每日签到，只有确认成功或已经签到才记录完成。
- 自动处理今日任务，并采用防浪费顺序：有次数先抽；没有次数时只完成一个加次数任务；奖励到账后立即抽，再判断下一个任务。
  - 模型使用任务达到要求后自动领奖，不会为了完成任务主动产生付费模型调用。
  - 每日答题优先复用本地已验证答案；遇到新题时记录错误选项并逐步完善答案缓存。
  - 站内广告任务按照服务端返回的时长正常等待，冷却结束后再领取，不绕过计时。
- 自动读取剩余翻牌机会并记录奖励、时间和累计次数。
- 守护模式同时考虑翻牌和今日任务冷却，按最早可执行时间安排下一轮。
- 支持余额显示、托盘常驻和 Windows 登录后自动启动守护。

## 运行环境

- Windows 10 或 Windows 11
- Node.js 18 或更高版本
- PowerShell 5.1 或更高版本
- Chrome 或 Microsoft Edge

## 安装

在 PowerShell 中运行：

```powershell
git clone https://github.com/wubaojing0129-pixel/vsllm-auto-checkin.git
cd vsllm-auto-checkin
npm install
```

如果当前电脑没有可用的浏览器内核，可以额外安装 Playwright Chromium：

```powershell
npx playwright install chromium
```

也可以双击 `Install-Dependencies.bat` 安装依赖。

## 首次使用

1. 双击 `VSLLM-Launcher.bat`。
2. 点击 `首次登录`。
3. 在弹出的 Chrome 或 Edge 窗口中登录 VSLLM。
4. 确认已经进入个人中心后，关闭登录浏览器窗口。
5. 回到登录命令窗口按 `Enter`，保存登录状态。
6. 在控制台点击 `签到+任务+翻牌一次`，或者点击 `开始守护`。

登录状态失效后，重新点击 `首次登录` 即可。

## 日常使用

| 按钮或选项 | 作用 |
| --- | --- |
| `首次登录` | 打开普通 Chrome/Edge 登录窗口并保存本机登录态。 |
| `签到+任务+翻牌一次` | 依次检查签到、今日任务，再按当前剩余机会翻牌。 |
| `开始守护` | 立即执行一轮，然后根据最早冷却时间自动安排下一轮。 |
| `停止守护` | 停止守护和当前后台任务。 |
| `刷新余额` | 只读取账户余额，不签到、不执行任务、不翻牌。 |
| `自动今日任务` | 控制是否自动领奖、答题和完成站内广告任务，默认开启。 |
| `抽奖记录` | 在控制台中查看最近抽奖历史。 |
| `打开记录文件` | 使用记事本打开完整抽奖历史。 |
| `清空窗口日志` | 只清空当前窗口，不删除历史记录。 |
| `清理旧日志` | 清理旧任务日志，不删除登录态、签到记录和抽奖记录。 |
| `启用开机守护` | Windows 登录后自动打开控制台并启动守护。 |

窗口最小化或点击右上角关闭时，程序会缩到系统托盘。要彻底退出，请右键托盘图标并选择 `退出程序`。

## 今日任务说明

### 模型使用任务

程序只会在网站显示任务进度已经达标时领取奖励。它不会替你调用付费模型，也不会为了凑任务次数消耗额度。

### 每日答题

程序把已验证的答案和排除过的错误选项保存在：

```text
logs/daily-task-answers.json
```

首次遇到新题时可能需要尝试和重试；学到正确答案后，相同题目会优先复用。单轮默认最多尝试 8 次，避免持续请求。

### 站内广告任务

程序调用网站正常的开始接口，按照服务端返回的时长等待，然后再领取奖励。如果奖励增加翻牌次数，程序会先确认当前没有可用次数，领取后立即翻牌，避免与容量上限冲突。如果任务处于冷却中，本轮会跳过，并由守护模式在更早的可用时间重新检查。

## 守护逻辑

守护模式启动后会立即执行一轮：

1. 打开个人中心并确认登录状态。
2. 检查每日签到并读取余额、翻牌机会和任务状态。
3. 先消耗当前已有的翻牌次数。
4. 没有可用次数时，只完成一个可执行任务。
5. 每次任务奖励到账后重新读取次数；有次数就立即翻牌，再决定是否执行下一个任务。
6. 达到单轮翻牌限制但仍有内容可执行时，约 1 分钟后继续。
7. 记录执行结果和下一轮时间。

下一轮会优先采用以下时间中最早的一个：

- 翻牌冷却结束时间
- 未完成广告任务的下一次可用时间
- 无法读取冷却信息时的兜底检查间隔

程序默认在预计刷新时间后额外等待约 10 秒，避免刚刷新时页面状态尚未更新。

## 翻牌次数

`每轮最多翻牌` 默认是 `3`，表示单轮最多请求几次，不是永久总次数限制。

例如网页或接口显示当前有 1 次机会，本轮只会翻 1 次；后续机会刷新后，守护模式仍会继续执行。

## 配置

控制台提供以下常用配置：

| 配置 | 默认值 | 说明 |
| --- | --- | --- |
| `自动今日任务` | 开启 | 自动检查并执行当前可完成的今日任务。 |
| `兜底间隔分钟` | `180` | 无法读取任何有效冷却时间时的备用检查间隔。 |
| `每轮最多翻牌` | `3` | 每轮最多请求几次翻牌。 |

高级用户也可以使用环境变量：

| 环境变量 | 说明 |
| --- | --- |
| `VSLLM_DAILY_TASKS` | 是否自动执行今日任务；设置为 `0`、`false`、`off` 或 `no` 可关闭。 |
| `VSLLM_TASK_AD_LIMIT` | 单轮最多完成几次站内广告任务，默认 `3`。 |
| `VSLLM_TASK_QUIZ_ATTEMPTS` | 单轮每日答题最多尝试次数，默认 `8`。 |
| `VSLLM_DRAW_LIMIT` | 每轮最多翻牌次数。 |
| `VSLLM_WATCH_INTERVAL_MINUTES` | 无法读取冷却时间时的守护兜底间隔。 |
| `VSLLM_WATCH_BUFFER_SECONDS` | 预计刷新后的额外等待秒数，默认 `10`。 |
| `VSLLM_USER_ID` | 自动识别 `new-api-user` 失败时手动指定用户 ID。 |
| `VSLLM_BROWSER_EXECUTABLE` | 指定 Chrome 或 Edge 可执行文件路径。 |
| `VSLLM_URL` | 指定个人中心地址，默认 `https://vsllm.com/console/personal`。 |

## 命令行使用

不使用图形控制台时，可以直接运行：

```powershell
npm run login:browser
npm run api
npm run api:balance
npm run api:watch
```

| 命令 | 作用 |
| --- | --- |
| `npm run login:browser` | 打开普通浏览器完成首次登录。 |
| `npm run api` | 执行一轮签到、今日任务和翻牌。 |
| `npm run api:balance` | 只读取余额。 |
| `npm run api:watch` | 启动命令行守护模式。 |
| `npm run api:headed` | 使用可见浏览器执行 API 流程，便于排查问题。 |

## 日志和本地数据

| 路径 | 内容 |
| --- | --- |
| `.auth/vsllm-profile/` | 本机登录状态，包含敏感信息，严禁上传。 |

## 故障排查

### 提示未登录或登录失效

点击 `首次登录` 重新登录。完成登录后关闭浏览器窗口，并回到登录命令窗口按 `Enter`。

### 提示首次登录窗口还没结束

先关闭登录浏览器窗口，再回到登录命令窗口按 `Enter`。如果找不到残留窗口，可以运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\stop-login-browser.ps1
```

### 守护没有立即执行下一轮

检查控制台日志中的下一轮时间。程序可能正在等待翻牌冷却、广告任务冷却或兜底检查时间。

### 顶部余额没有刷新

点击 `刷新余额`。如果仍然无法读取，可能是网站接口发生变化，或者登录状态已经失效。

### 控制台没有显示新代码效果

运行中的控制台不会自动加载新脚本。请在系统托盘右键选择 `退出程序`，再重新打开 `VSLLM-Launcher.bat`。
